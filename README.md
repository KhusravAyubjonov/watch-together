# Хусрав ♡ Азизашка — Watch Together

Романтическая приватная комната для совместного просмотра трансляции с двух телефонов. Сайт статический и подходит для GitHub Pages; видео не проходит через сервер приложения.

## Возможности

- персональная ссылка-приглашение без регистрации и PIN-кода;
- YouTube, прямые `.m3u8` (HLS) и `.mp4` ссылки;
- автоматическое встраивание официальной страницы `https://ftv.tj/live.html`;
- синхронные Play, Pause и перемотка;
- прямой P2P-чат и присутствие в реальном времени через PeerJS;
- романтические реакции и адаптивный интерфейс для телефона;
- ссылка на внешний источник, если сайт запрещает встраивание.

## Соединение двух устройств

По умолчанию чат, реакции и синхронизация работают напрямую между двумя браузерами через PeerJS. Ничего настраивать не требуется: хозяин комнаты должен держать страницу открытой, пока Азизашка подключается.

### Необязательно: Supabase Realtime

Supabase можно подключить как альтернативный транспорт.

1. Создайте проект на [supabase.com](https://supabase.com/).
2. Откройте **Project Settings → API**.
3. Скопируйте `Project URL` и публичный `anon` key.
4. Вставьте их в `config.js`:

```js
window.WATCH_TOGETHER_CONFIG = {
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_ANON_KEY"
};
```

Таблицы создавать не требуется: приложение использует временные Broadcast-каналы Supabase Realtime. Проверьте в **Realtime Settings**, что Realtime включён.

## Локальный запуск

Файлы нельзя полноценно проверять двойным кликом из-за ограничений браузера. Запустите любой статический сервер, например:

```powershell
npx serve .
```

## GitHub Pages

В настройках репозитория откройте **Pages**, выберите **Deploy from a branch**, ветку `main` и каталог `/ (root)`. После публикации главная страница создаёт ссылку формата:

```text
https://khusravayubjonov.github.io/watch-together/#/room/...
```

Хозяин должен оставаться в комнате, когда гость впервые подключается: источник передаётся напрямую через Realtime и не сохраняется в общей базе.

## Ограничения источника

Обычная ссылка на страницу другого сайта не всегда является ссылкой на видео. Для встроенного воспроизведения источник должен предоставить YouTube embed, прямой HLS/MP4 URL, корректный CORS и не использовать DRM. Приложение не обходит защиту сторонних сервисов.
